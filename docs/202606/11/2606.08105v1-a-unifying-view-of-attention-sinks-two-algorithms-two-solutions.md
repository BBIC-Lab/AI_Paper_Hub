## 摘要
当注意力集中于单个token，即一个汇聚点时，模型实际上在计算什么？注意力汇聚在softmax Transformer中无处不在，然而这种共同的视觉特征可能掩盖了根本不同的算法。我们表明，视觉上相似的汇聚模式可以反映两种不同的机制：(i) 自适应空操作，其中头通过路由到空token来抑制其更新，以及 (ii) 广播，其中汇聚点聚合并重新分发全局信息。在这种情况下，汇聚点起着类似的作用：当没有有用的计算时，一个安全的目的地。提出的干预措施如门控或寄存器之所以有效，是因为它们隐含地针对其中一种机制，揭示了方法和假设机制之间的二元性：门控隐含地假设空操作；寄存器隐含地假设广播。每种机制都留下独特的痕迹（空操作汇聚点表现出可忽略的值范数；广播汇聚点产生低秩输出），我们在合成任务中对其进行了形式化，并用来推导实用的诊断方法。应用于预训练视觉Transformer，这些诊断表明两种机制在大规模下都存在：汇聚点从早期层的CLS token转移到更深层的图像块，并集中在专门的头上。引人注目的是，原本为广播设计的寄存器token被重新用于服务于空操作，证实了单一干预措施不够。将门控与寄存器结合在稳定性和性能方面产生了互补的增益。总的来说，我们发现相同的注意力模式可以反映两种非常不同的计算，有效的干预需要首先询问模型实际在计算什么。

## Abstract
When attention concentrates on a single token, a sink, what is the model actually computing? Attention sinks are ubiquitous in softmax transformers, yet this shared visual signature can hide fundamentally different algorithms. We show that visually similar sink patterns can reflect two distinct mechanisms: {i} adaptive nop, where a head suppresses its update by routing to a null token, and {ii} broadcast, where a sink aggregates and redistributes global information. In that case, sinks serve an analogous role: a safe destination when there is nothing useful to compute. Proposed interventions like gating or registers work because they implicitly target one or the other, revealing a duality between method and assumed mechanism: gating implicitly assumes nop; registers implicitly assume broadcast. Each mechanism leaves distinct traces (nop sinks exhibit negligible value norms; broadcast sinks induce low-rank outputs) which we formalize on synthetic tasks and use to derive practical diagnostics. Applied to pretrained vision transformers, these diagnostics reveal that both mechanisms exist at scale: sinks transition from CLS in early layers to patches in deeper layers, and concentrate in specialized heads. Strikingly, register tokens, designed for broadcast, are repurposed to also serve nop, confirming that neither intervention alone suffices. Combining gating with registers yields complementary gains in stability and performance. Overall, we find that the same attention pattern can reflect two very different computations and effective intervention requires first asking what the model is actually computing.

- **整体含义**：论文提出一个二元假设框架，将注意力汇聚区分为**自适应 NOP（空操作）** 和**广播**两种算法机制，并给出可测量的诊断标志，以此揭示预训练 Vision Transformer 中两种机制共存，且单一干预（寄存器或门控）不足以覆盖所有情况。最后证明组合干预能带来互补增益。

## 2. 方法论
- **核心思想**：将注意力汇聚视为两种计算原语：  
  - **自适应 NOP**：头将注意力集中在某个“空 token”上，其值向量范数极小，从而实现对残差流的近似零更新，硬开关由大奇异值或 massive activation 实现。  
  - **广播**：汇聚 token 的值向量携带全局信息，添加到多个 token 后使输出矩阵近似秩-1，并增加 token 间相似性。
- **关键诊断指标**：  
  1. **汇聚 token 的值范数比值** $\frac{\|v_s\|}{\|v\|}$：远低于其他 token 即暗示 NOP。  
  2. **注意力更新矩阵的稳定秩（stable rank）**：接近 1 时暗示广播行为。  
  3. **Q‑K 交互矩阵的谱尖峰**：用于检测硬开关。  
- **合成验证任务**：  
  - **NOP 任务**：要求位于特定方向（flag）的 token 输出被衰减（$\gamma=0$ 即为完全 NOP），其他 token 保持原样。  
  - **广播任务**：所有 token 都需添加某个指定的源 token 的旋转变换，强制全局信息共享。  
  在两个合成任务上训练最少层的 Transformer 块，观察是否自发涌现对应的汇聚模式以及上述诊断标志，并用正则化系数、门控概率做扫参分析。

## 6. 主要结论与发现
- 视觉上相同的注意力汇聚条纹可能是**自适应 NOP**（值范数极低）或**广播**（秩-1 更新、值范数正常），两种机制在同一预训练 ViT 中共存，并随层化转移：早期层以 [CLS] 为 sink，深层转为图像块。
- 寄存器 token 原为广播设计，但在实际模型中也被重新用于 NOP，说明单一干预无法覆盖两种机制。
- 组合使用门控注意力（针对 NOP）和寄存器（支撑广播），在稠密预测任务（ADE20K、VOC）上取得互补增益，而 ImageNet 分类变化不大；这表明缓解汇聚主要改善的是局部空间表征，而非全局分类能力。

## 7. 优点
- **概念清晰**：将“相同的注意力条纹”分解为两个可验证的计算假设，并提供可操作的诊断工具，避免仅凭注意力图做出误判。
- **合成任务驱动**：通过最小合成任务精确复现两种汇聚模式，为诊断指标的合理性提供了因果证据。
- **实证全面**：在多个大规模 ViT 上验证双机制共存，并展示了合并干预带来的实际性能提升，为模型设计提供指导。
- **框架通用性强**：诊断指标（值范数；稳定秩）不依赖特定架构，可推广到其他 Transformer 模型。

## 8. 不足与局限
- **实验范围**：缓解实验仅在一个自监督训练任务（LeJEPA）和一种规模的 ViT 上完成，泛化到其他视觉任务或语言模型的有效性待验证。
- **仅关注视觉 Transformer**：结论和诊断指标主要在 ViT 上验证，对语言模型的适用性未深入实验，尽管文中提及 LLM 中的 sink 现象。
- **组合干预并非根除广播汇聚**：门控+寄存器虽然抑制了 NOP 类汇聚，但广播汇聚仍然存在（只是被引导到寄存器），这可能在某些任务中仍引入低秩偏好。
- **缺少对训练动态的深层分析**：未探究为什么优化器会自发发现并分离这两种汇聚原语，也未分析不同数据分布或训练长度的影响。
