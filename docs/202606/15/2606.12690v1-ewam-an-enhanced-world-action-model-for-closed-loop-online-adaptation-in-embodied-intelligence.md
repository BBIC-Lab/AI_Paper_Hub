## Abstract
In this paper, we propose the Enhanced World Action Model (EWAM), a closed-loop online adaptation architecture built upon a pretrained and fully frozen Cosmos3 backbone network. Evaluated entirely under a zero-shot task protocol, EWAM is centrally focused on reducing the amount of additional deployment data required to adapt to new task layouts. Notably, no extra task-specific demonstration sets were introduced in any of the evaluations, and no fine-tuning was performed on the backbone network. Its performance gains stem entirely from an inference-time co-reasoning mechanism composed of four inserted lightweight neural layers: the Neural Experience Memory Layer located in the intermediate layers of the Diffusion Transformer (DiT) provides task-relevant execution context; the Neural Anomaly Detection Layer after the state prediction head monitors the divergence between predicted and actual states in real time; the Neural Policy Routing Layer dynamically selects direct execution, conservative replanning, or rollback recovery based on the anomaly severity; and the Neural Action Correction Layer refines the generated action chunks using execution diagnostics. Unlike naive feature fusion, the memory, anomaly detection, and correction modules are deeply integrated into the Cosmos3 forward path in a differentiable manner, with only the final routing decision being a discrete supervised one.

## 1. 论文核心问题与整体含义

- **核心问题**：具身智能机器人在开放环境中部署时面临巨大挑战。收集覆盖多样物体姿态、场景布局和执行意外情况的大量演示数据成本极高。现有的世界行动模型（WAM）虽然结合了未来状态预测和动作生成，但其静态的离线推理流程容易受到执行层面的不匹配影响，如碰撞、空抓取和感知幻觉，且无法高效地将成功经验转化为后续的行动优势。
- **整体含义**：论文旨在解决WAM模型在**零样本（Zero-Shot）部署到新任务时，如何进行高效在线自适应的问题**，核心目标是**减少对额外任务演示数据和骨干网络微调的依赖**。EWAM的提出，表明通过插入轻量级、可训练的神经层来增强冻结的预训练主干网络，可以在推理时实现闭环自适应，从而提升执行质量和效率。

## 2. 方法论

- **核心思想**：将在线自适应视为一个**推理时闭环问题**。EWAM保持一个预训练好的Cosmos3-Nano--Policy-DROID主干网络（参数 $\phi$）完全冻结，通过在其前向传播路径中插入多个可微的神经层（参数 $\theta$）来实现动态的自适应能力，其性能提升完全来自这些模块在推理时的协同推理。
- **关键技术细节**：EWAM架构在Cosmos3主干网络（包含视觉语言编码器、自回归推理器和扩散变换器DiT）中插入了四个关键神经层：
    1.  **神经经验记忆层（Neural Experience Memory Layer）**：插入在DiT的中间层（$l_{mem}=6$）。它从记忆库 $M$ 中检索与当前任务相关的历史成功经验，并通过注意力机制将检索到的上下文注入到DiT的隐藏状态中。
        $$h_{DiT}^{(l_{mem})} = h_{DiT}^{(l_{mem}-1)} + \alpha_{mem} \cdot \text{Attention}(q_{mem}, k_m, v_m)$$
    2.  **神经异常检测层（Neural Anomaly Detection Layer）**：位于状态预测头之后。它结合了上一轮预测残差、当前候选动作的动力学一致性，以及由DiT隐藏状态学习到的碰撞、空抓取、幻觉和过力等风险概率，生成一个6维的异常向量 $\iota_t$。
        $$\iota_t = (e_{prev}, e_{dyn}, \iota_{collision}, \iota_{grasp}, \iota_{halluc}, \iota_{force})$$
    3.  **神经策略路由层（Neural Policy Routing Layer）**：位于异常检测之后。它根据异常向量 $\iota_t$、推理上下文 $c_t$ 和记忆状态，通过一个MLP和Softmax函数输出离散的路由决策 $r_t$，动态选择**直接执行（direct）**、**保守重规划（conservative）** 或**回滚恢复（rollback）**。
    4.  **神经动作校正层（Neural Action Correction Layer）**：位于动作输出头之后。它根据路由决策 $r_t$ 执行不同操作：如果是回滚，则从记忆库恢复一个安全动作；如果是保守规划，则使用一个轻量网络 `CorrectionNet` 对原始动作 $a_{t:t+H}^0$ 进行细化，生成最终执行的校正动作 $a_{t:t+H}^*$。
        $$a_{t:t+H}^* = a_{t:t+H}^0 + \alpha_{corr} \cdot \text{CorrectionNet}(a_{t:t+H}^0, \iota_t, M)$$
- **闭环在线学习与过滤**：系统实时检测执行失败（碰撞、空抓取等），并触发回滚或保守重规划。一个**质量门过滤器**会阻止不安全的、低效的轨迹进入经验记忆库和用于在线参数更新，防止策略被污染。更新仅局限于插入的神经层参数，确保了预训练主干网络的知识不被破坏。

## 3. 实验设计

- **仿真平台**：所有实验均在RoboLab高保真仿真环境中进行。
- **基准任务（Benchmark）**：评估了三种零样本操控任务，旨在测试对不同形态物体的泛化能力：
    1.  `BananaInBowlTask`（软物体容器操控）
    2.  `BananasInBinOneMoreTask`（带干扰物的容器转移）
    3.  `BlockStackingOrderAgnosticTask`（多物体顺序无关堆叠）
- **对比方法**：
    1.  **基线模型**：**Cosmos3-Nano--Policy-DROID**，即未加载EWAM层的冻结主干网络。
    2.  **消融实验模型**：在`BananaInBowlTask`上对比了7种EWAM变体，分别移除了**经验记忆（w/o Memory）**、**自学习（w/o Self-Learning）**、**错误校正（w/o Error Correction）**、**回滚（w/o Rollback）**，以及仅使用**记忆（Memory-only）** 和仅使用**参数更新（Update-only）** 的版本。

## 4. 资源与算力

- **硬件平台**：所有计算在一台配置了两块 **NVIDIA RTX 5880 Ada** 显卡（每块48GB显存）的工作站上进行。
- **推理开销**：在单块GPU上，EWAM相比Cosmos3-DROID基线（66.2ms）增加了 **15.4%** 的推理延迟，总计 **76.4ms**。额外耗时主要来自记忆层（+3.1ms）和校正层（+2.9ms）。
- **训练细节**：论文未提及主干的预训练过程或EWAM模块的离线训练总时长，但明确了在线更新的学习率为 $10^{-4}$，适配器秩为16，回放缓冲区大小为1000。

## 5. 实验数量与充分性

- **实验数量**：进行了核心任务对比（3个任务）和全面的消融研究（7个变体）。每项实验均采用 **5个随机种子**，每个种子进行 **25次试验**，即每项任务每模型经过 **125次评估**，整个三任务子集共进行 **375次评估**。
- **充分性与公平性**：实验设计较为充分和公平。
    - **一致性**：所有对比模型共享相同的冻结主干网络和动作接口。
    - **严格对比**：评估序列和扰动条件（物体位置和光照）对所有模型保持一致，由固定种子控制。
    - **统计严谨**：结果报告了 **95%置信区间**，并通过配对统计检验进行分析。
    - **局限性**：实验仅覆盖RoboLab仿真中的3个任务，任务多样性有限。

## 6. 主要结论与发现

- **效率大幅提升**：在`BananaInBowlTask`上，EWAM在保持100%成功率的同时，相较Cosmos3-DROID，任务完成时间**减少63.8%**（25.60s -> 9.27s），路径长度**缩短54.1%**（1.81m -> 0.83m），总执行故障**降低83.7%**（13.5 -> 2.2）。
- **关键模块贡献明确**：消融实验表明，**在线自学习**对性能贡献最大，移除后任务时间增加402%。**经验记忆层**和**动作校正层**是关键模块，移除后任务时间分别增加14.4%和96.3%。
- **记忆优先策略有效**：“记忆优先，适配器次之”的设计被证实有效。仅使用记忆检索（12.67s）的表现优于仅使用参数更新（14.67s），验证了从零样本稀疏数据中检索历史经验比立即修改预训练权重要更稳定。
- **泛化能力初显**：在多任务泛化测试中，EWAM在更复杂的堆叠任务上不仅提升了效率，还将成功率从87.2%提升至94.4%，展示了其应对复杂操作的潜力。

## 7. 优点

- **新颖的架构思路**：提出了一种“冻结主干+可微推理层”的优雅架构，将在线自适应问题巧妙转化为模型推理的一部分，避免了成本高昂且风险较大的微调。
- **模块化与可解耦性**：记忆、异常检测、路由、校正四个模块职责清晰，可单独训练和组合，便于理解和改进。
- **高效的零样本自适应**：在完全不引入新任务演示数据和更新主干网络的情况下，仅通过闭环反馈实现了显著的性能提升，验证了其“数据高效”和“部署高效”的特性。
- **完善的故障处理机制**：通过异常检测、策略路由和质量过滤，构建了一个从故障预警、决策响应到避免错误学习的完整闭环，显著提升了执行的安全性。

## 8. 不足与局限

- **仿真到真机的鸿沟**：所有验证均在RoboLab仿真环境中完成，未进行任何真实机器人实验，其回滚恢复等机制在真实世界的有效性未知。
- **任务场景有限**：仅在两类共三个操控任务上进行了验证，尚未在长时域、复杂动态交互等更严苛的场景中进行测试。
- **硬件条件要求**：文中报告的推理延迟（76.4ms）超过了典型伺服环路（如20Hz, 50ms预算）的要求，可能需要异步推理或控制器解耦，对高频实时控制构成挑战。
- **门槛与泛化性**：六个经验过滤阈值是依据特定任务场景设定的，迁移到新任务时可能需要重新标定。其架构也专门为Cosmos3主干设计，对其他WAM或VLA模型的通用性有待考证。
