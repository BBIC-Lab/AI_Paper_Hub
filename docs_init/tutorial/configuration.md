# 订阅与查询配置

订阅决定系统每天关注什么主题，查询决定系统如何从论文池中召回候选。好的配置应该既能覆盖你的核心方向，又不会把太多无关论文带进精读区。

## 配置目标

- 把研究兴趣拆成几个稳定主题。
- 为每个主题写出可复用的查询描述。
- 设置精读区和速读区的候选上限。
- 保存后能通过日报结果反向调参。

## Step 1：先写清楚研究主题

不要一开始就堆关键词。建议先用一句话描述你真正想跟踪的问题，例如：我想每天看到和多模态医学影像诊断相关的新论文，优先关注数据集、模型架构和临床验证。

然后把它拆成三个层次：

1. 核心方向：必须出现或高度相关的主题。
2. 扩展方向：可以接受的相邻主题。
3. 排除方向：容易误召回但你不想看的主题。

## Step 2：新增或编辑订阅

1. 打开修改查询或订阅管理页面。
2. 新增一个订阅，填写清晰的名称。名称建议短一些，方便日报中识别来源。
3. 填写自然语言查询描述，让系统知道你想找什么，而不是只给零散关键词。
4. 如果页面提供关键词、排除词或来源限制，按第一步的拆分结果填写。

检查点：保存后刷新页面，订阅仍能正常显示。

## Step 3：设置精读区与速读区上限

精读区用于保留最值得深看的论文，速读区用于保留可能有价值但优先级稍低的论文。建议先从较小数量开始，例如每个查询精读 5 到 10 篇、速读 5 到 10 篇，再根据日报质量调整。

调参原则：

- 精读区太吵：降低精读上限，增加排除词，或收紧查询描述。
- 漏掉相关论文：增加速读上限，放宽扩展方向，观察几天结果。
- 每日报告太长：优先降低速读区，再考虑降低精读区。

## Step 4：保存并触发一次验证运行

1. 保存配置。
2. 回到 GitHub Actions，手动触发一次日报工作流。
3. 打开运行日志，确认查询配置被正确读取。
4. 运行完成后，打开最新日报，检查每篇论文的主题来源、推荐理由和分区。

检查点：日报里的推荐理由能对应你的订阅描述，而不是只匹配到个别泛词。

【截图占位：Actions 日志中读取订阅配置的位置】

## Step 5：根据结果迭代配置

第一次配置很难完美，建议连续观察三天。每天只改一到两个变量，这样能判断是哪项修改带来了变化。

| 结果表现 | 优先调整 | 示例动作 |
| --- | --- | --- |
| 相关论文少 | 查询描述和扩展方向 | 增加同义词、任务名、数据集名 |
| 无关论文多 | 排除词和精读上限 | 排除高频误召回主题，降低精读数量 |
| 推荐理由空泛 | 查询描述 | 写清楚你关心的方法、应用场景和评价标准 |
| 每天重复内容多 | 来源和时间窗口 | 检查论文源同步，避免过宽的历史窗口 |
